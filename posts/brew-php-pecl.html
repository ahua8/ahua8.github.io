Macos brew安装多版本php,pecl安装扩展问题,在5.6与7.2之间总是安装好一个,另一个版本扩展被删除了

September 14, 2018 - 1 minute read -
macos pecl 多版本PHP

至从2018.03.31起, Homebrew PHP扩展从brew tap 中移除了,没有再维护. 并且PHP版本只有5.6跟7.0+,默认php 安装的是 PHP7.2, 好在这些版本中的 PHP都自带安装了pecl, 到是省了不少事, pecl install xdebug 也算方便.

但是在安装多版本时, pecl安装也就开始出问题了.

安装PHP信息

安装完成后,可以看见生成pecl的配置文件

这两个目录就是存放扩展.so文件的地址, /usr/local/lib/php/pecl/20170718 与 /usr/local/lib/php/pecl/20131226

`/usr/local/opt/php@5.6/bin/pecl config-show` 与 `/usr/local/opt/php/bin config-show` 如果ext-dir 目录是相同的.就会下面这种问题
在5.6下 /usr/local/opt/php@5.6/bin/pecl install xdebug-2.5.5 安装的扩展后, 再在7.2下 /usr/local/opt/php/bin/pecl install xdebug 安装同样的扩展, 会提示已安装. 如果 uninstall 后, 再 install. 则
/usr/local/lib/php/pecl/20131226 下的的 xdebug.so 会被删除.而20170718会新安装一条.
不会两个目录同时存在

问题在哪呢?

pecl config-show 时,你会发现倒数第2行.

User Configuration File        Filename         `/Users/aaron/.pearrc`
这是一个缓存起来的 pear.conf文件, pecl 命令会优先加载此文件.而不是/usr/local/etc/php/5.6/pear.conf, /usr/local/etc/php/7.2/pear.conf, 导致,两个版本的 PHP 共用了一个配置.

解决方案:

删除配置User Configuration File 项中的文件.

rm -rf /Users/aaron/.pearrc 
再去安装就可以不冲突了.
可以通过 pecl list 查看已安装的扩展
